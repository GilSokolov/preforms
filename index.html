Preforms

A class-based form modeling system for Angular

Build forms as real domain models — not configuration objects.

Preforms gives you a strongly-typed form modeling system with a built-in trigger engine for cross-field validation, behavior and a domain-ready field library.

Why Preforms

Most Angular form solutions treat forms as configuration:

  • JSON schemas
  • Config objects
  • Template-driven mappings

Preforms takes a different approach:

Forms are object models with behavior.

You define forms using classes like FormField, FieldGroup, NumberField, and compose them like a real domain model.

No manual form wiring is required — models are interpreted into reactive forms and UI components.

1. Form Modeling System

Define forms using strongly-typed classes.

new NumberField({ key: 'amount' })
new InputField({ key: 'username' })

2. Trigger Engine

Add cross-field behavior directly inside field definitions.

new PasswordField({
  key: 'confirmPassword',
  triggers: [
    {
      on: 'input',
      action: TriggerAction.VALIDATE,
      validation: {
        operator: 'eq',
        compareTo: 'password',
        message: 'must match password'
      }
    }
  ]
})

3. Domain Field Library (DX layer)

Preforms ships with a rich set of domain-ready field primitives.

Semantic Fields
new EmailField()
new PasswordField()
new PhoneNumberField()

These fields include sensible defaults, structure, and validation.

Domain Fields
new Address()
new Currency()
new DateOfBirth()
new FullName()

Smart Defaults vs Explicit Control

Preforms balances minimal configuration with explicit identity.

Intent-based fields (no key required)

new EmailField()
new PasswordField()

Data-bound fields (explicit identity required)

new NumberField({ key: 'amount' })
new InputField({ key: 'username' })

Extensible UI System

You can create your own strongly-typed fields.

export class AmountField extends NumberField {
  constructor() {
    super({
      key: 'amount',
      label: 'Amount',
      min: 0
    });
  }
}

Preforms allows custom field components to be plugged directly into the form model.

Custom Field Components

Preforms lets you create fully custom UI field components.

@DynamicFormElement({
  component: 'password-strength',
})
export class PasswordStrengthComponent extends BaseFieldComponent<
  FormControl<number>,
  PasswordStrengthField
> {}
Bind it to a field model
export class PasswordStrengthField extends NumberField {
  constructor() {
    super({
      key: 'strength',
      component: 'password-strength',
    });
  }
}

What makes Preforms different

  • Strongly typed form models
  • Object-oriented field system
  • Built-in cross-field validation
  • Declarative trigger engine
  • No JSON schema required
